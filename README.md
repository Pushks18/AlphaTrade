# ⚡ ComputeX

> **Decentralized GPU Compute × AI Model Ownership × Autonomous Trading**

ComputeX is a decentralized platform built on 0G Chain. It enables users to rent GPU compute power, train AI models offchain, trustlessly mint those models as **ERC-7857 iNFTs** (Intelligent NFTs), and deploy them as autonomous trading agents using KeeperHub and Uniswap.

---

## 🏗️ Architecture overview

1. **Smart Contracts (`/ComputeX-Contracts`)**:
   - `GPUMarketplace.sol`: Escrow for renting GPU time.
   - `ModelNFT.sol`: Implements ERC-7857 to mint AI models into NFTs. Pulls a "mint right" atomically from the GPU Marketplace to prevent double-minting.
   - `ModelMarketplace.sol`: Secondary market to trade trained Model NFTs.
2. **Offchain Backend (`/backend`)**:
   - Listens to onchain events.
   - `train.py`: Trains the AI model (simulated linear trend follower for MVP).
   - `prove.py`: Generates a cryptographic proof of the training (zkML stub).
   - `upload.ts`: Uploads model weights and proofs to 0G Storage.
   - `agent.py`: Fetches the model, runs inference on live price data, and executes trades via KeeperHub to Uniswap.
3. **Frontend (`/frontend`)**:
   - Next.js dApp providing a rich UI to list GPUs, rent them, mint NFTs, and trade models.

## 🚀 Step-by-Step Local Setup

### 1. Start the Local Blockchain (Anvil)
Open a terminal and start Anvil to run a local EVM network:
```bash
anvil
```
*Leave this running in the background.*

### 2. Deploy the Smart Contracts
Open a new terminal and deploy the contracts to your local Anvil instance:
```bash
cd ComputeX-Contracts
forge script script/Deploy.s.sol:Deploy --rpc-url http://127.0.0.1:8545 --broadcast --private-key 0xac0974bec39a17e36ba4a6b4d238ff944bacb478cbed5efcae784d7bf4f2ff80
```
> Note: The private key above is the default `Account 0` provided by Anvil.

### 3. Start the Backend Orchestrator
The backend listens for events from the contracts and runs the AI pipeline.
```bash
cd backend
npm install
npm run orchestrator
```
*Leave this running in the background.*

### 4. Start the Frontend dApp
Open a new terminal and start the Next.js frontend:
```bash
cd frontend
npm run dev
```
The dApp will be available at [http://localhost:3000](http://localhost:3000) (or port `3001` if `3000` is busy).

### 5. Configure MetaMask
1. Open MetaMask and add a custom network:
   - **Network Name**: Local Anvil
   - **RPC URL**: `http://127.0.0.1:8545`
   - **Chain ID**: `31337`
   - **Currency Symbol**: `ETH`
2. Import one of the private keys generated by Anvil (e.g., `0xac0974bec39a17e36ba4a6b4d238ff944bacb478cbed5efcae784d7bf4f2ff80`) into MetaMask so you have test ETH to pay for transactions.

## 🎬 Example Workflow (End-to-End Walkthrough)

Open the frontend at [http://localhost:3000](http://localhost:3000) and connect your wallet. Then follow these steps to experience the entire lifecycle:

### Phase 1: List a GPU (Provider)
1. Go to the **🖥 GPU Market** tab.
2. Enter your hardware specs URI (e.g., `ipfs://QmGpuSpecs...`).
3. Set an hourly rental price (e.g., `0.001` ETH).
4. Click **List GPU →**.
5. Once confirmed, note your new **GPU ID** (e.g., `0`).

### Phase 2: Rent Compute & Train Model (Renter)
1. Go to the **⚙️ Compute Jobs** tab.
2. Enter the **GPU ID** from Phase 1.
3. Set the duration (e.g., `2` hours) and click **Rent GPU & Start Job →**.
4. **Watch the backend terminal!** The orchestrator will detect the `JobCreated` event and automatically:
   - Run `train.py` to train the AI model.
   - Run `prove.py` to generate the zkML proof stub.
   - Upload the weights and proof to 0G Storage (simulated locally if API key is not set).
   - Wait for the provider to complete the job.
5. In the frontend **Job Inspector**, enter your Job ID (e.g., `0`) and click **Complete Job (Provider)** to release the escrow.
6. **Watch the backend terminal again!** The orchestrator will now automatically call `ModelNFT.mintModel()` onchain using the returned storage CIDs.

### Phase 3: Inspect the Model NFT
1. Go to the **🧠 Model NFTs** tab.
2. In the **NFT Inspector**, enter Token ID `1` and click **Fetch**.
3. You will see the ERC-7857 iNFT metadata decoded directly from the blockchain, including the `modelCID` and `proofCID` linked to your freshly trained AI model!

### Phase 4: Trade the Model
1. Go to the **🛒 Trade Models** tab.
2. Under **List Model for Sale**, enter Token ID `1` and set a price (e.g., `0.05` ETH). Click **Approve & List →**.
3. (Optional) Switch to a different wallet in MetaMask to act as a buyer.
4. Under **Buy Model NFT**, enter Token ID `1`, click **Lookup**, then click **Buy Model NFT →**. The smart contract automatically routes a 5% royalty to the creator and a 2.5% fee to the protocol!

### Phase 5: Autonomous Trading Agent
1. On the **🛒 Trade Models** tab, locate the **Run Trading Agent** section at the bottom.
2. Click **Run Agent →**.
3. The frontend will simulate loading the model from 0G Storage, running an inference on live synthetic price data, and outputting a `BUY` or `SELL` signal.
4. It will then simulate submitting the swap execution to Uniswap via KeeperHub!
